# Инструкции проекта (PROJECT_RULES.md)

Используй эти правила при анализе кода, написании функций и исправлении ошибок в проекте CUSTOM-CLI-CNOTE.

## 1. Стиль и стандарты TypeScript

- Используй строгую типизацию (TypeScript). Избегай типа any.

- Предпочитай интерфейсы (interface) для определения структур данных.

- Соблюдай именование в стиле camelCase для функций и переменных.

- Всегда добавляй JSDoc-комментарии для публичных методов, объясняя их назначение, параметры и возвращаемые значения.

## 2. Анализ и отладка (Check-bugs Protocol)

При выполнении команды /ask @PROJECT_RULES.md анализ ...:

1. **Поиск узких мест:** Проверяй наличие потенциальных утечек памяти или неэффективных циклов.

2. **Безопасность:** Ищи небезопасные методы работы с файловой системой или аргументами командной строки.

3. **Обработка ошибок:** Убедись, что все асинхронные операции (async/await) обернуты в try/catch с адекватным логированием.

4. **Валидация:** Проверяй входные данные (аргументы CLI) перед их использованием.

## 3. Структура CLI

- Учитывай, что проект является CLI-утилитой.

- Старайся разделять логику обработки аргументов (парсинг) и логику выполнения действий.

- Вывод в консоль должен быть структурированным (используй цвета или символы для обозначения успеха/ошибок, если это возможно).

## 4. Исключения

- Не включай в анализ содержимое папок node_modules, dist, .git.

- При написании кода всегда учитывай ограничения Node.js среды.

## Как использовать этот файл:

- В чате Continue пиши: /ask @PROJECT_RULES.md Проанализируй этот файл @cnote.ts на предмет ошибок.

- Агент сам прочитает эти правила и применит их к выбранному контексту.
